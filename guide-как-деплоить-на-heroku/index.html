Guide: Как деплоить на heroku?

В интернете огромное количество гайдов по деплою, но мало кто указывает важные моменты, различные упрощения и фишечки, поэтому я вам все покажу. В качестве примера я буду использовать скрипт на языке python, который репостит видео из каналов на youtube в группу вконтакте:

Для начала, нам нужно зарегистрироваться на heroku.com. Если не знаете что вписать в поле Role - пишите Hobbyist. Затем, установите приложение heroku для командной строки; я подразумеваю, что вы уже занимаетесь разработкой и git уже установлен.

В принципе, вы сможете обойтись и без heroku cli, но так вы сможете смотреть логи вашего приложения, использовав команду heroku logs в нужной папке либо heroku logs -a <имя приложения> находясь в любой директории.

Теперь, мы переходим на наш dashboard и создаем приложение. Конечно, можно использовать heroku cli для создания приложений, но есть одна сложность: вам нужно будет не забыть указать имя приложения (либо переименовать его позже в веб-настройщике приложений или опять же через heroku cli) и флаги для расположения сервера, т.к. он будет по умолчанию расположен в штатах, что создаст некоторый пинг при работе с европейскими серверами.

Для деплоя нам нужен будет скачаный локальный репозеторий git. Начнем:

Создаем ветку heroku:

git branch heroku
git checkout heroku

В нашем бранче создаем файлы:

Procfile:
worker: python ваш_скрипт.py (учтите, worker - это именно ваш алиас,
                             вы можете выбрать любое удобное вам название)

Runtime:
python-3.6.6 (совместимая или последняя версия питона)

requirements.txt:
Список библиотек, каждая с новой строки.

И, если надо, настраиваем локальный конфиг.

Указываем пути для загрузки репозетория на сервер heroku:

git remote heroku https://git.heroku.com/<имя приложения>.git

Вы так-же можете посмотреть текущие удаленные пути репозетория через команду git remote -v или удалить не нужные/ошибочные через команду git remote remove <имя пути>.

Коммитим наши изменения и проталкиваем их на сервер heroku:

git push heroku heroku:master

Важным замечанием являеться то, что проталкивать надо на целевую ветку мастера, иначе просто ничего не произойдет.

Теперь осталось включить в веб-меню приложений наш воркер нажав на значок карандаша и перетянув тумблер в право. Готово, мы запустили приложение на удаленном сервере!

Но если вам нужно сделать запуск приложения по расписанию, а не еденично или использовав его в качестве сервиса - надо подключить аддоны для расписания. Я пробовал пользоваться двумя:

  1. Heroku Scheduler - его минусом являеться не гибкость в выборе промежутока времени: выполнение раз в 10 секунд, в час или в сутки.
  2. Process Scheduler - не официальный аддон от heroku с гибким графиком запуска или работы вашего приложения. Из минусов можно отметить, что если мы не можем использовать скрипт одиночного запуска, надо переделать его в сервис, допустим обернув в Flask или использовав while(true): wait.sleep(1), как сделал я.

И конечно, вы можете смотреть логи.
